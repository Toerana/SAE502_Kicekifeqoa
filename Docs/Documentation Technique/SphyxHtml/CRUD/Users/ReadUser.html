Read User¶

Ce module permet de récupérer des informations sur les utilisateurs, les groupes auxquels ils appartiennent, ainsi que les tâches et sous-tâches qui leur sont assignées via des requêtes API.

Configuration de l’API¶

L’URL de l’API est définie comme suit :

url = "https://kicekifeqoa.alwaysdata.net/api.php"

Fonctions¶

get_data(table, columns='*', filter_column=None, filter_value=None, join_table=None, join_condition=None)¶

Récupère des données génériques d’une table avec des options de filtrage et de jointure.

Paramètres:

  • table (str) – Le nom de la table à interroger.

  • columns (str, optionnel) – Les colonnes à récupérer (par défaut “*” pour toutes les colonnes).

  • filter_column (str, optionnel) – La colonne utilisée pour le filtrage.

  • filter_value (str, optionnel) – La valeur utilisée pour le filtrage.

  • join_table (str, optionnel) – La table à joindre.

  • join_condition (str, optionnel) – La condition de jointure.

Renvoie:

Un dictionnaire JSON contenant les résultats ou un message d’erreur.

Type renvoyé:

dict ou str

—

get_users(id_user=None, email=None)¶

Récupère des informations sur les utilisateurs.

Paramètres:

  • id_user (int, optionnel) – L’ID de l’utilisateur à rechercher.

  • email (str, optionnel) – L’email de l’utilisateur à rechercher.

Renvoie:

Une liste de dictionnaires contenant les informations des utilisateurs ou “no existing links” si aucune correspondance n’est trouvée.

Type renvoyé:

list ou str

Exemple d’utilisation :

users = get_users(id_user=1)
print(users)

—

get_users_group(id_user=None)¶

Retourne les groupes auxquels appartient un utilisateur.

Paramètres:

id_user (int, optionnel) – L’ID de l’utilisateur à rechercher.

Renvoie:

Une liste de dictionnaires contenant l’ID de l’utilisateur, l’ID du groupe et le nom du groupe.

Type renvoyé:

list ou str

Exemple d’utilisation :

groups = get_users_group(id_user=3)
print(groups)

Exemple de sortie :

[{'id_user': 3, 'group_id': 2, 'name': 'IT'}]

—
