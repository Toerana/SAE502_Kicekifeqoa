Delete Subtask¶

Ce module permet de supprimer des sous-tâches en utilisant une API PHP et une connexion MySQL.

Configuration de l’API¶

L’URL de l’API est définie comme suit :

url = "https://kicekifeqoa.alwaysdata.net/api.php"

Configuration de la Connexion MySQL¶

config = {
    'user': '379269_admin',
    'password': 'Kicekifeqoa123*',
    'host': 'mysql-kicekifeqoa.alwaysdata.net',
    'database': 'kicekifeqoa_todolist',
}

Fonctions¶

close_connection_BDD(conn, cursor)¶

Ferme proprement la connexion à la base de données et affiche un message de confirmation.

Paramètres:

  • conn (mysql.connector.connection.MySQLConnection) – L’objet de connexion à la base de données.

  • cursor (mysql.connector.cursor.CursorBase) – Le curseur de la base de données.

Exemple d’utilisation :

close_connection_BDD(conn, cursor)

Exemple de sortie :

La connexion à la base de données a été fermée.

—

delete_subtask(table, column, value)¶

Supprime une sous-tâche en envoyant une requête DELETE à l’API.

Paramètres:

  • table (str) – Le nom de la table où se trouve la sous-tâche (par exemple, « Subtask »).

  • column (str) – Le nom de la colonne utilisée pour identifier la sous-tâche (par exemple, « id_subtask »).

  • value (str) – La valeur de l’identifiant pour la suppression.

Renvoie:

Affiche la réponse JSON de l’API et ferme la connexion à la base de données.

Type renvoyé:

None

Exemple d’utilisation :

delete_subtask("Subtask", "id_subtask", "1")

Exemple de post_data envoyé à l’API :

{
    "table": "Subtask",
    "column": "id_subtask",
    "value": "1"
}

Exemple de sortie en cas de succès :

{"status": "success", "message": "Subtask deleted successfully"}

Exemple de sortie en cas d’erreur :

{"status": "error", "message": "Subtask not found"}
