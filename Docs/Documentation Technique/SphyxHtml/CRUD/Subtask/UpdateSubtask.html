Update Subtask¶

Ce module permet de mettre à jour des sous-tâches en utilisant une API PHP.

Configuration de l’API¶

L’URL de l’API est définie comme suit :

url = "https://kicekifeqoa.alwaysdata.net/api.php"

Fonctions¶

update_subtask(id_subtask, id_affected_task=None, name=None, end_date=None, checked=None)¶

Met à jour une sous-tâche en envoyant une requête POST à l’API.

Paramètres:

  • id_subtask (int) – L’ID de la sous-tâche à mettre à jour.

  • id_affected_task (int, optionnel) – L’ID de la tâche principale associée à la sous-tâche.

  • name (str, optionnel) – Le nouveau nom de la sous-tâche.

  • end_date (str, optionnel) – La nouvelle date d’échéance au format “YYYY-MM-DD HH:MM:SS”.

  • checked (int, optionnel) – Le nouvel état de vérification (1 pour vérifié, 0 pour non vérifié).

Renvoie:

Un message indiquant le succès ou l’échec de la mise à jour.

Type renvoyé:

str

Exemple d’utilisation :

result = update_subtask(
    id_subtask=12,
    name="Nouvelle Sous-Tâche",
    end_date="2024-10-20 15:30:00",
    checked=1
)
print(result)

Exemple de post_data envoyé à l’API :

{
    "table": "Subtask",
    "action": "update",
    "data": {
        "name": "Nouvelle Sous-Tâche",
        "end_date": "2024-10-20 15:30:00",
        "checked": 1
    },
    "column": "id_subtask",
    "value": 12
}

Exemple de sortie en cas de succès :

"Sous-tâche avec ID 12 mise à jour avec succès."

Exemple de sortie en cas d’erreur :

"Erreur lors de la mise à jour : 400 - Bad Request"

En cas d’erreur de connexion à l’API :

"Erreur de connexion à l'API : [détails de l'erreur]"
